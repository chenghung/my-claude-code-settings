## 觸發條件

每當修改了以下任何一項設定後，必須立即執行本規則所定義的完整檢視流程：

- `skills/` 目錄下的任何檔案
- `agents/` 目錄下的任何檔案
- `rules/` 目錄下的任何檔案
- `.claude/rules/` 目錄下的任何檔案
- `CLAUDE.md`

## 檢視項目

### 上下文規則衝突檢查

確認新修改的內容與現有的其他 rules、agent 定義、skill 定義之間，沒有互相矛盾的指令或行為要求。任何衝突都必須在完成本次修改前解決。

### 幻覺風險檢查

確認所有 prompt 中沒有引用不存在的工具、不存在的檔案路徑、不存在的功能，或描述與實際行為不符的內容。發現任何此類問題，必須立即更正。

### 重複 prompt 檢查

找出跨檔案之間是否存在重複或高度相似的 prompt 片段。重複的指令會浪費 token，且可能在不同上下文中產生不一致的行為，必須合併或刪除冗餘內容。

### 安全憑證檢查

嚴格禁止將任何敏感的安全憑證直接置於 `skills/`、`agents/`、`rules/` 或 `.claude/rules/` 目錄下的任何檔案中。

禁止放入的內容包含但不限於：API 金鑰、密碼、存取權杖（access token）、含有 token 的 webhook URL、私密金鑰（private key）、任何形式的 secret。

此類資訊必須透過環境變數或外部的 secrets 管理機制來處理，不得以明文形式寫入版本控制的檔案中。

若在檢視過程中發現既有檔案包含上述內容，必須立即向使用者提出警告，並建議以安全的替代方案取代。

## 檢視範圍

執行檢視時，必須涵蓋以下所有相關檔案：

- `rules/` 目錄中的全部 `.md` 檔案
- `.claude/rules/` 目錄中的全部 `.md` 檔案
- `agents/` 目錄中的全部 agent 定義檔案
- `skills/` 目錄中的全部 skill 定義檔案
- 根目錄與 `.claude/` 目錄下的 `CLAUDE.md`

## 執行要求

若在上述任何檢視項目中發現問題，必須在完成本次修改前主動提出並修正，不得將問題遺留至後續作業。
