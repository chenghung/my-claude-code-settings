# 暫存檔案存放規則

任務期間若需建立暫存檔，必須遵守以下規則。

## 存放位置

- 優先使用工作區根目錄下的 `.tmp` 資料夾；該資料夾由使用者或專案決定是否建立，LLM 不得主動建立。
- 僅當工作區確認沒有 `.tmp` 資料夾，或使用者明確要求時，才可改用系統 `/tmp` 目錄。

## 判斷 .tmp 是否存在

判斷工作區根目錄下的 `.tmp` 資料夾是否存在時，必須以檔案系統層級的檢查為準，例如使用 `test -d <workspace>/.tmp`。不得以 git 追蹤狀態（例如 `git ls-files`、`git status` 的輸出）作為判斷依據，因為 `.tmp` 可能已被加入 `.gitignore` 而未被 git 追蹤，但實際存在於檔案系統中。此外，`test -d` 會自動 follow symlink，即使 `.tmp` 是 symlink 也能正確判斷。

## 任務結束後的處理

- 任務完成後應主動清除不再需要的暫存檔；工作區內的暫存檔不得提交至版本控制。
